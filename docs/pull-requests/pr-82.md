# [PR 82](https://github.com/vig-os/commit-action/pull/82) ci: add published-tag consumer smoke workflow

Closes #81.

## What

A new project-owned workflow, `published-tag-smoke.yml`, that runs the **released artifact from its git tag** — the way a consumer pins it (`uses: vig-os/commit-action@vX.Y.Z`) — and asserts it produces a **signed** commit with the expected content. Sibling of `e2e-smoke`, which only exercises `uses: ./` from a branch checkout and so never covers the published tag ref after it is pushed.

## Why it targets `main` directly

`workflow_dispatch` is only dispatchable when the workflow lives on the **default branch**. This is a deliberate, scoped deviation from the dev-first flow so we can trigger it for the already-pushed `v0.3.0-rc1`; `sync-main-to-dev` carries it back to `dev` for future releases. It is a brand-new file, so it does not touch the in-flight `release/0.3.0` PR (#78) or the devkit-managed release workflows.

## Design notes

- **Triggers:** `push: tags: ['v*.*.*']` (matches RCs `v0.3.0-rc1` and finals `v0.3.0`; excludes the floating `v0`/`v0.X` tags that move on promotion) **and** `workflow_dispatch` with a `ref` input.
- **Fires on tag push** because the release pipeline pushes tags with the **Release App** token (not `GITHUB_TOKEN`), so tag-push events trigger downstream workflows.
- **`uses:` forbids expressions**, so the action is checked out **at the tag** into a path and run via `uses: ./action-under-test` — a faithful reproduction of a consumer pinning `@<tag>`.
- **Assertions mirror `e2e-smoke`:** commit exists → is `verification.verified == true` → content round-trips; scratch branch deleted on exit. No recursion (scratch push is a branch, not a tag).

## Validating this release

The `push` trigger will not retro-fire for the already-pushed `v0.3.0-rc1`. Once this merges to `main`, dispatch it manually against `v0.3.0-rc1`:

```
gh workflow run published-tag-smoke.yml --ref main -f ref=v0.3.0-rc1
```

Local checks green: `actionlint` clean, `just precommit` (prek --all-files, incl. yamllint + shellcheck) all pass.
