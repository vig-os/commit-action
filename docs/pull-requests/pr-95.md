# [PR 95](https://github.com/vig-os/commit-action/pull/95) ci: drop top-level contents write from smoke workflows

## What

Restrict the default `GITHUB_TOKEN` to none at workflow scope in `e2e-smoke.yml` and `published-tag-smoke.yml`, declaring `contents: write` on the single job instead (needed for the scratch-branch push and the commit the action under test creates via the API). Same layout as `codeql.yml` / `scorecard.yml` / `sync-issues.yml`.

## Why

Scorecard's TokenPermissions check scores 0 whenever any workflow grants a write permission at workflow scope — these two were the only remaining top-level writes. Fixing them closes code-scanning alerts #27 and #23 and lifts the check score off zero. The remaining job-level write alerts in the release pipeline are tracked separately (out of scope here; several of those files are devkit-managed).

## Verification

- Full pre-commit suite green locally (`prek run --all-files`).
- `actionlint` clean for this diff (the two `steps.commit.outputs` warnings it reports are pre-existing on `dev`, unrelated).
- `e2e-smoke` runs on this PR (pull_request → dev) — exercises the job-level token end-to-end.
- `published-tag-smoke` to be dispatched on this branch with `ref=v0.3.0` after the PR opens.
