Closing in favour of **#58** (migrate action to ESM) — **note this PR is green, and that is precisely the problem.**

`@actions/http-client` v4 is also **ESM-only**. It passes CI only because **CI never bundles**: the workflows run `just sync` / `just precommit` / `just test` and never call `just bundle`. `npm run bundle` fails on it exactly as it does on #49 and #50.

Merging it would leave `dev` in a state where `dist/` **cannot be rebuilt** — and that failure would surface later, in the release cycle, on a change CI had already reported as safe.

Held rather than merged, deliberately. It lands together with #49 and #50 as part of #58.
