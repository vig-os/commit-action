# [PR 108](https://github.com/vig-os/commit-action/pull/108) build(devkit): adopt devkit 1.4.0

## Summary

Re-scaffold this repo from **vigOS devkit 1.3.1 → 1.4.0** (direnv mode) via the
installer (`--force --no-prompts --mode direnv`; values resolved from the
existing `.vig-os` manifest, mode/identity preserved).

This PR **pins `1.4.0-rc2` to validate it ahead of the final `1.4.0` release**;
the pins will be bumped to the final `1.4.0` revision before merge. Pins move in
lockstep:

| Pin | Value |
|-----|-------|
| `.vig-os` `DEVKIT_VERSION` | `1.4.0-rc2` |
| `flake.nix` `vigos.url` | `github:vig-os/devkit?ref=1.4.0-rc2` |
| `flake.lock` vigos rev | `6fe5f7704cc4d9d7894ff003ecdbd77746bf6ec7` (= tag `1.4.0-rc2^{}`) |

Preserved from `.vig-os`: `DEVKIT_MODE=direnv`, `DEVKIT_TAG_PREFIX=v`,
`DEVKIT_FLOATING_TAGS=major,minor`.

## Verified change signatures

**vig-os/devkit#1182 — zizmor workflow audit + baseline (Security)**
- `persist-credentials: false` on the read-only checkouts (`ci.yml`
  resolve-toolchain/lint, `codeql.yml`, `renovate-changelog-build.yml`,
  `sync-issues.yml`).
- `sync-main-to-dev.yml` cleanup step: release-app token moved into `env:`
  (`RELEASE_GH_TOKEN`) — template-injection fix.
- New devkit-owned `zizmor.yml` baseline (scaffolded), so this repo carries no
  baseline of its own.

**vig-os/devkit#1173 — self-hosted CI runner routing**
```diff
+ runner-json:  # resolve-toolchain output, JSON array from DEVKIT_CI_RUNNER
- runs-on: ubuntu-24.04
+ runs-on: ${{ fromJSON(needs.resolve-toolchain.outputs.runner-json) }}
```
New empty `.vig-os` key `DEVKIT_CI_RUNNER=` (absent ⇒ hosted `ubuntu-24.04`).

**vig-os/devkit#1180 — forward flake `shellHook` env in direnv CI**
(`setup-devkit-toolchain/action.yml`).

**vig-os/devkit#1181 — `with-native-libs` base `justfile` recipe** for `uvx`
tools with native wheels.

**vig-os/devkit#1170 — `pymarkdown` flake system hook** (flows via the
flake-generated hook set, direnv mode).

## Consumer customizations preserved

- Project-owned files untouched: `.github/workflows/release-extension.yml`,
  `justfile.project`, `renovate.json` (including the TypeScript-major Renovate
  hold, Refs #101).
- `codeql.yml` language matrix / push paths unchanged (Node globs).

## Verification

- `just precommit` (flake-generated hook set, `hooksExcludes = [ "^dist/" ]`)
  runs fully green locally.
- Note: the installer re-writes a hand-managed `.pre-commit-config.yaml` on
  re-scaffold even for flake-hooks consumers; it is gitignored (never committed)
  and removed post-scaffold so the flake symlink takes over — same manual step
  as the 1.3.1 adoption. Not a 1.4.0 regression (identical in 1.3.1).
