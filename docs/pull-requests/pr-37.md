# [PR 37](https://github.com/vig-os/commit-action/pull/37) ci(sync): check out the triggering ref, not dev, in the sync job

## Unblock `Sync main to dev`

`dev` is stuck 4 commits behind `main` and cannot catch up: the sync workflow has
been failing on every push to `main` since #32.

Failing run: https://github.com/vig-os/commit-action/actions/runs/29314999211

### The bug

The `sync` job checks out `dev`, then invokes a **local** composite action:

```yaml
      - name: Checkout dev
        with:
          ref: dev                                        # ŌåÉ workspace is now dev's tree
      - name: Set up devkit toolchain
        uses: ./.github/actions/setup-devkit-toolchain    # ŌåÉ resolved from dev's tree
```

GitHub resolves `uses: ./ŌĆ”` against the **checked-out workspace**, not the
workflow's ref. `.github/actions/` landed on `main` with the devkit 1.1.0 adoption
(#32) and does not exist on `dev`, so the step dies:

```
Can't find 'action.yml', 'action.yaml' or 'Dockerfile' under
'ŌĆ”/.github/actions/setup-devkit-toolchain'.
```

`resolve-toolchain` and `check` pass because they check out the **default** ref
(`main`), where the action does exist. Only `sync` pins `dev`.

This is a **bootstrap deadlock** ŌĆö the only thing that would carry the action onto
`dev` is the very sync PR this job can no longer open. It cannot self-heal.

### The fix

Drop `ref: dev`, so the workspace is the triggering `main` SHA where the action is
guaranteed to exist.

Nothing downstream reads dev's **working tree**. Every subsequent step already
operates on remote refs or the API:

| Step | Operates on |
|---|---|
| Re-check if dev is still behind main | `git fetch origin main dev` ŌåÆ `git rev-list --count origin/main ^origin/dev` |
| Detect merge conflicts | `git merge-tree --write-tree origin/dev origin/main` |
| Create sync branch from main | `git checkout -b "$SYNC_BRANCH" origin/main` |
| Existing-PR check ┬Ę stale-branch cleanup ┬Ę Create PR ┬Ę auto-merge | `gh` / REST API |

`fetch-depth: 0` and the commit-app token are unchanged, so merge-base and push
behavior are identical. The job now builds against the *newer* tree ŌĆö the one whose
actions it actually references.

### Upstream

This file is the devkit scaffold template **verbatim** (`diff` against
`assets/workspace/.github/workflows/sync-main-to-dev.yml` on `devkit@main` is
empty), so the defect ships to every scaffolded repo. The devkit's own copy has the
same pattern with `./.github/actions/setup-env` ŌĆö latent, one rename away from
firing.

Filed upstream as **vig-os/devkit#1034**. This PR is the **local** fix so
development can continue without waiting on a devkit release. A header comment and
a step comment mark the divergence so a future re-scaffold does not silently revert
it.

### Verification

- `just precommit` fully green locally (incl. `yamllint --strict`, `check-yaml`).
- Behavioral proof is the next push to `main`: merging this PR itself triggers
  `Sync main to dev`, which should now open the `chore/sync-main-to-dev-*` PR that
  finally brings `.github/actions/` onto `dev`.
