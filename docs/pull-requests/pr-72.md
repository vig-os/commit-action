# [PR 72](https://github.com/vig-os/commit-action/pull/72) ci(dist): scope Dist Check to release/** and main PRs

Closes #71 (follow-up to #59).

Drops `dev` from `dist-check.yml`'s triggers so Renovate runtime-dep bumps no longer fail on a stale bundle — nothing ships from `dev`. By design, staleness inherited from `dev` now surfaces on the `release/X.Y.Z → main` draft PR right after `prepare-release` (Dist Check still runs there) and is fixed by a bugfix PR to the release branch; `release-extension.yml`'s finalize-time verification remains the hard tag guarantee.

Already applied separately: Dev protection ruleset required checks reduced to `CI Summary` (done before this PR so it wouldn't hang on an expected-but-never-reported check).
