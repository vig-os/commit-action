# [PR 93](https://github.com/vig-os/commit-action/pull/93) ci: remove the Dist Check bootstrap stub

## What

Remove `.github/workflows/dist-check-stub.yml` (the temporary always-green `Dist Check (bootstrap stub)`).

## Why

The stub existed only to satisfy the required `Dist Check` status context on branches whose head predated the real freshness gate (before 0.3.0). Now that **0.3.0 is promoted to main**, the real `dist-check.yml` (#59) is on `main` and `dev` and reports the same contractual `Dist Check` context. Per the stub's own lifecycle note, leaving it in place would **mask a genuine failure** of the real gate (both workflows report the same context, and the always-green stub would keep the check green). So it must go.

Safe to remove: the real `dist-check.yml` job `name: Dist Check` matches the required context exactly, so the ruleset requirement stays satisfied.

Closes #85
