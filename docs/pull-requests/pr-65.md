# [PR 65](https://github.com/vig-os/commit-action/pull/65) build(npm): migrate ESLint to flat config and bump to v10

## Migrate ESLint to flat config + v10

Closes #64. Supersedes Renovate's **#53**, which could not work as a bump.

### Why a bump wasn't enough

ESLint 9 made flat config the default; **ESLint 10 removed the legacy `.eslintrc`
path entirely**, and `ESLINT_USE_FLAT_CONFIG=false` no longer opts back in:

```
ESLint couldn't find an eslint.config.(js|mjs|cjs) file.
```

There was a second blocker until recently: `@typescript-eslint@6.21.0` **hard-crashed**
under ESLint 10 (its bundled `eslint-scope@7.x` yields a scope manager that ESLint 10's
`SourceCode.finalize()` rejects):

```
TypeError: scopeManager.addGlobals is not a function
```

**#57 cleared that** by bumping typescript-eslint to `8.64.0`, whose peer range is
`eslint: ^8.57.0 || ^9.0.0 || ^10.0.0`. So the plugin side was already ready; only the
config format was left.

### What changed

- `.eslintrc.json` в†’ **`eslint.config.mjs`**. Faithful port, no intended behavior change:
  same base configs (`eslint:recommended` + typescript-eslint `recommended`), the same
  two rule overrides (`no-explicit-any: warn`, `explicit-function-return-type: off`), and
  the `node`/`es6`/`jest` `env`s re-expressed as `languageOptions.globals` (flat config has
  no `env`).
- `eslint` `8.57` в†’ `10.7.0`.
- `@typescript-eslint/eslint-plugin` + `@typescript-eslint/parser` в†’ the **`typescript-eslint`**
  meta package. This is the canonical v8 flat-config entry point and composes the
  `eslint-recommended` rule-disables correctly, instead of us hand-wiring them. Both
  packages remain installed transitively.
- Adds `@eslint/js` and `globals` (the flat-config equivalents of the `extends` and `env`
  keys that went away).

Node 24 satisfies ESLint 10's engines (`^20.19.0 || ^22.13.0 || >=24`).

### Verification

- `npx eslint src` вЂ” lints **all 8** source files, rules firing correctly.
- **Negative control** вЂ” injected `const __probe: any = 1; let __unused = __probe;` into
  `src/commit.ts` and confirmed ESLint caught it (`prefer-const`,
  `@typescript-eslint/no-unused-vars`, `no-explicit-any`), then reverted. The config is not
  silently linting nothing.
- `tsc` clean В· **84/84 tests pass** В· `just precommit` fully green.
- `dist/` untouched.

### вљ пёЏ Found while doing this: `npm run lint` has never linted the real source

Not introduced here, and **not fixed here** вЂ” filed separately.

The lint and format scripts use `src/**/*.ts`. npm runs scripts through `sh`, which has no
globstar, so `**` degrades to `*` and the pattern collapses to `src/*/*.ts`:

```console
$ sh -c 'echo src/**/*.ts'
src/__tests__/setup.ts
```

**One file.** `src/commit.ts`, `src/commit-runner.ts`, and `src/retry.ts` have never been
linted or formatted, and the "Lint & Format" CI job has been passing on that basis.

Running ESLint over the real tree surfaces **29 errors and 17 warnings across 8 files**, and
`prettier --list-different src` reports **all 8 files** as unformatted. Most errors are
`no-require-imports` in test mocks (defensible in tests, needs a rule override). The only two
in production source are minor and **neither is a correctness bug**:

- `src/commit-runner.ts:177` вЂ” `preserve-caught-error` (no `cause` on a rethrow; new v10 rule)
- `src/commit.ts:85` вЂ” `no-useless-assignment` (dead initializer on `bytesRead`; harmless)

I deliberately kept that out of this PR: fixing the glob means triaging 46 findings and
reformatting the entire source, which has a different risk profile and would make both changes
unreviewable. This PR's ESLint 10 support is verified directly via `npx eslint src` regardless
of the script bug.
