# [PR 31](https://github.com/vig-os/commit-action/pull/31) fix(devcontainer): fail loudly when tsx is unavailable in post-create

## Summary

`.devcontainer/scripts/post-create.sh` masked tsx install failures and could report a false success. The tsx step ran `npm install -g tsx >/dev/null 2>&1` then printed `tsx installed (<version>)` using a probe that falls back to `""` on failure. On the new hermetic Nix image the global npm prefix lands off-`PATH` (vig-os/devcontainer#728), so `tsx` was **not** actually available, yet the script printed `tsx installed ()` and exited 0.

## Changes

- **Stop swallowing the critical install.** Drop `>/dev/null 2>&1` and check the real exit status of `npm install -g tsx`; exit non-zero with a clear message if the install command itself fails.
- **Verify tsx is actually reachable after install.** If `command -v tsx` succeeds, use it. Otherwise fall back to the pinned local devDependency via `npx --no-install tsx` (tsx is already in `devDependencies`). If neither is reachable, exit non-zero with an actionable message instead of reporting false success.
- **Document the apt/NodeSource fallback.** Add a comment noting it is a Debian-only path, guarded by `command -v node`/`npm` (so it is skipped on the Nix image where Node is baked in). Full removal is tracked separately under the devcontainer bump (#29) and node-24 work (#14/#28) — not ripped out here.

Behaviour is unchanged on the current Debian image (`ghcr.io/vig-os/devcontainer:0.1`) where the global tsx is on `PATH`.

## Verification

`shellcheck -x` and `bash -n` pass. Simulated all four code paths with stubbed `tsx`/`npm`/`npx`:

| Scenario | Result |
|---|---|
| tsx on PATH (current Debian image) | prints version, **exit 0** |
| global install off-PATH, local devDep reachable via npx (Nix image) | prints `... (via npx)`, **exit 0** |
| global install off-PATH AND npx cannot find tsx | clear error, **exit 1** |
| `npm install -g tsx` itself fails | clear error, **exit 1** |

## Context

Found while live-testing the vigOS devcontainer Nix migration. Related upstream image fixes: vig-os/devcontainer#727 (`/usr/bin/env`) and vig-os/devcontainer#728 (npm global prefix off-PATH).

Closes #30
