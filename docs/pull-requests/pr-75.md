# [PR 75](https://github.com/vig-os/commit-action/pull/75) fix(npm): lint and format the whole source tree, not one file

Closes #66.

`npm run lint` and `npm run format` have never checked the real source. Both globbed with `src/**/*.ts`, but npm runs scripts through `sh`, which has no globstar — so `**` degrades to a single `*` and the pattern collapses to `src/*/*.ts`, matching only files exactly one directory deep. In practice: one test-setup file. `commit.ts`, `commit-runner.ts` and `retry.ts` — the whole production surface — were matched by neither script.

Fixed by passing the directory (`eslint src`, `prettier --write src`) and letting each tool expand its own globs. eslint now checks **6 files instead of 1**.

## Four commits, deliberately separate

**1. `fix(npm)` — the script fix + the two production findings.** Linting the real tree surfaced two, both fixed here, neither a correctness bug:

- `commit-runner.ts` rethrew a `git status` failure without attaching the original error, discarding the underlying `execSync` cause (`preserve-caught-error`).
- `isBinaryFromStat` initialised `bytesRead` to `0` and always overwrote it before reading (`no-useless-assignment`). Dropping the initialiser also lets the compiler's definite-assignment analysis see the read — which it now proves, so the change is verified rather than assumed.

**2. `style(src)` — the whole-tree prettier reformat.** Mechanical, no behavior change; isolated so it can be reviewed or dropped independently.

Note what it actually applies: the repo's own checked-in `.prettierrc` (`singleQuote: true`, `printWidth: 100`), which **the source has never matched** — it's all double-quoted at ~80 cols, precisely because `format` never ran on it. So most of the ~609/765-line diff is the resulting quote flip and re-wrapping, not logic.

**3. `build(dist)` — rebuild the bundle.** Picks up commits 1–2. Needed because the e2e smoke job executes the *committed* bundle, not the source.

**4. `ci(lint)` — gate PRs on eslint and prettier.** The root cause was never just the glob: **nothing ran these tools at all.** The scaffold-managed `ci.yml` has a "Lint & Format" job, but it runs `just precommit` — the prek hook suite, which contains no JS linter or formatter. That is how the broken globs survived for the life of the repo, and fixing the scripts without a gate would let them rot again the same way.

Adds a consumer-owned `js-quality` workflow (a separate file, since `ci.yml` is managed and must not be edited — the same arrangement `dist-check.yml` uses). It runs `just lint` and a new read-only `just format-check` recipe, i.e. the same recipes a developer runs locally, so a green CI means a green tree rather than a CI-only invocation that can drift.

## Scope note

The issue also catalogued ~25 `no-require-imports` findings and planned a `src/__tests__` rule override for them. **Both are now moot**: the ESM migration (#58) removed `require()` from the test suite entirely. What was a 46-finding triage is down to the two above.

## Verification

- **`ESLint & Prettier` — the new gate ran on this PR and passed**, executing `eslint src` and `prettier --check src` for real (`All matched files use Prettier code style!`), not vacuously.
- **`E2E Smoke` passed**, so the rebuilt bundle still performs a genuine signed commit against the live API — the reformat and both source fixes are verified end to end, not on faith.
- `npm test` — 84/84 passing. `npx tsc --noEmit` — 0 errors. `just precommit` — green.
