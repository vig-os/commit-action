# [PR 32](https://github.com/vig-os/commit-action/pull/32) build(devkit): adopt vigOS devkit 1.1.0 (direnv) — rollout pilot

## Adopt the vigOS devkit 1.1.0 (direnv) — first rollout pilot

First real deployment of the mode-aware devkit (1.1.0) to a consumer repo, in
**direnv** mode. This is the pilot for the wider vigOS rollout.

### What changed
- **Replaced the legacy apt/Debian `.devcontainer/`** with the devkit 1.1.0
  direnv scaffold (Nix flake + `.envrc`, mode-aware CI, shared standards kit).
  Pruned via `--prune-devcontainer`, which removes the old `post-create.sh`
  (the Debian-only script from #30).
- **Wired the Node toolchain** for this JS action: `nodejs` in the flake
  dev-shell + npm-mapped `justfile.project` recipes (`sync`/`lint`/`test`/
  `build`/`bundle`). Build tools (tsc, jest, ncc, tsx, eslint, prettier) come
  from `npm ci`; nixpkgs provides node + npm.
- **Opted into the flake-generated pre-commit config** (`hooks = { }`), so hooks
  use flake-sourced tools — fixes the `typos` hook, which previously downloaded
  a glibc binary that cannot run on NixOS. Adds commit-message validation +
  branch-naming enforcement.
- **Restored Node/Action `.gitignore` entries** the Python-only scaffold dropped
  (`node_modules/`, `*.tsbuildinfo`, `coverage/`) and removed the blanket
  `dist/` ignore so the committed `dist/index.js` bundle stays tracked.

### Verification (local, NixOS, direnv)
- `just sync/lint/test/build/bundle` all pass — **80 jest tests green**, tsc
  compiles, ncc bundles `dist/index.js`.
- `just precommit` **fully green** (typos now via the flake; `no-commit-to-branch`
  satisfied by this branch name).
- Dev-shell delivers node v24.16.0 + npm 11.13.0 + the devkit toolchain with no
  Nix eval warnings.

### Notes / follow-ups (not in this PR)
- The action **runtime stays `node20`** (`action.yml`); the dev-shell builds with
  node24, which is fine for the pure-JS, ncc-bundled output. Runtime migration is
  tracked separately (#28, #14).
- **Full release flow not adopted yet:** base is `main` (no `dev` branch); the
  scaffolded release / `sync-main-to-dev` workflows assume a `dev` branch. Adopt
  the dev→main flow as a follow-up.
- Devkit gap found during this pilot and filed upstream:
  [`vig-os/devkit#1024`](https://github.com/vig-os/devkit/issues/1024) — scaffold
  `.gitignore` is Python-only (worked around here).
- Pre-existing Dependabot findings on `main` (npm deps) are untouched here.

Closes #29
