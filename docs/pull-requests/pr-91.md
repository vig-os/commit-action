# [PR 91](https://github.com/vig-os/commit-action/pull/91) ci(release-core): sync deps before building the release artifact

## What

Add `just sync` before `just bundle` in the `Build release artifact` step of `release-core.yml`, on `release/0.3.0`.

## Why

The 0.3.0 release rolled back: `Finalize Release Core → Build release artifact` exited **127** because `just bundle` → `npm run bundle` → `ncc …`, and `ncc` (`@vercel/ncc`) is a devDependency only present after `npm ci`. The devkit toolchain provisions the dev-shell but not `node_modules`, so the bundler was missing from PATH. See RCA in #90 and the failed [run](https://github.com/vig-os/commit-action/actions/runs/29423806600).

`just sync` (language-neutral: `npm ci` here) installs the deps first, matching every other build job.

## Note — stopgap

`release-core.yml` is devkit-scaffolded. This is a local stopgap so 0.3.0 can be re-run **without waiting for a re-scaffold**. The durable fix is upstream in vig-os/devkit#1131 (CI green, merging into devkit `dev`); a future `DEVKIT_VERSION` bump will supersede this diff.
