# [PR 33](https://github.com/vig-os/commit-action/pull/33) build(action): migrate action runtime to node24

## Summary

Moves the action to the **node24** runtime ahead of GitHub removing the Node 20 Actions runtime on **2026-09-16** (Node 24 became the default 2026-06-02).

Closes #14. (Duplicate #28 was closed and consolidated here.)

## Changes
- `action.yml`: `runs.using` `node20` → `node24`
- `package.json`: `@types/node` `^20.10.0` → `^24.0.0` (no `engines.node` field exists; TypeScript 5.9.x resolves fine against `@types/node@24`)
- `tsconfig.json`: `target`/`lib` `ES2020` → `ES2023` (appropriate for the Node 24 baseline)
- Rebuilt committed `dist/` (tsc + ncc); `package-lock.json` refreshed
- `CHANGELOG.md`: `## Unreleased` entry

## Verification
- `npm run build` (tsc), `npm run bundle` (ncc), `npm test` → **80/80 pass**, 3 suites
- Pre-commit suite green (run in the devkit Nix dev-shell)

## Caveats / reviewer notes
- **Local build node was v22, not 24** — code is 24-compatible (builds against `@types/node@24` + ES2023, all tests pass) but true runtime-24 execution should be confirmed by CI on a node24 runner.
- ESLint is **not** in the pre-commit gate here; a pre-existing lint failure (test-file `no-var-requires`, present on `main`, no `src/` changes in this PR) is out of scope — worth a separate cleanup issue.
- After merge: **tag a release** so downstream consumers (e.g. `vig-os/devcontainer`) pick up node24.
