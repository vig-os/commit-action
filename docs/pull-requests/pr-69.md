# [PR 69](https://github.com/vig-os/commit-action/pull/69) ci(dist): gate PRs and releases on a fresh dist/ bundle

Closes #59.

Two commits:
1. `build(dist): rebuild bundle` — **dev's committed bundle was stale** (built before the undici 6.27.0 bump), which is exactly the drift this issue targets. Rebuilt with the pinned dev-shell toolchain; reviewer-verified reproducible (independent rebuild produces zero diff).
2. The gate itself:
   - New consumer-owned `dist-check.yml` (PRs to `dev`/`release/**`/`main`): rebuild with the devkit toolchain, fail if the committed `dist/index.js` differs. Job name `Dist Check` is contractual (required check, #60).
   - `release-extension.yml` no-op replaced with release-time verification of `inputs.finalize_sha` — a stale bundle fails the `extension` job and triggers release.yml's existing rollback. `workflow_call` inputs contract unchanged.

Deviation from the issue text, deliberate: the gate diffs `dist/index.js` (the sole runtime artifact, build-location independent) rather than `dist/` — the tree also carries 32 accidentally-committed tsc byproducts whose `.d.ts.map` files embed absolute paths and would make a whole-tree diff permanently red in CI. Cleanup of that cruft is a separate follow-up.

Negative test: injected marker into `dist/index.js` → gate exits 1; restored → 0. This PR also exercises the positive path via its own `Dist Check` run.
