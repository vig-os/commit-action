Closing — **blocked on the upstream ecosystem, not on anything in this repo.**

Nothing published today supports TypeScript 7:

- `ts-jest@29.4.11` (latest) peers `typescript` at `<7`
- `typescript-eslint@8.64.0` (latest) peers `typescript` at `<6.1.0`

So neither the test runner nor the linter can run under TS 7, whatever we do here. Confirmed against the registry, not assumed.

Two things found while diagnosing, for when this comes back:

**1. This PR was DOA anyway.** Renovate bumped `package.json` but never regenerated `package-lock.json` — its own `npm install` hit the peer conflict above — so `npm ci` failed before lint or tests ever ran:

```
npm error Invalid: lock file's typescript@5.9.3 does not satisfy typescript@7.0.2
```

That is why all three checks went red, not anything to do with TypeScript itself.

**2. A real breaking change waits behind it.** `tsconfig.json:12` uses `moduleResolution: "node"`, which TS 7 removed outright:

```
tsconfig.json(12,25): error TS5108: Option 'moduleResolution=node10' has been removed.
```

That same line is on the critical path for the ESM migration (#58), which replaces it with `node16`/`bundler`. So #58 landing first makes a future TS 7 bump substantially easier.

Renovate will re-propose this when the ecosystem catches up.
