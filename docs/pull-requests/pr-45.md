Renovate has already retitled this PR *"- abandoned"*, having detected the version is satisfied; closing it explicitly rather than leaving it to age out.

**Why it was batched rather than merged.** Renovate opens one PR per npm package, and every one of them touched both `package-lock.json` and `CHANGELOG.md`. Once #41 merged, all six went `CONFLICTING/DIRTY` against `dev` *and* against each other — landing them serially would have meant six rounds of rebase → regenerate lockfile → re-run CI → merge, with each merge re-dirtying the rest. #57 regenerated the lockfile once off `dev` instead: one CI run, one merge.

Nothing is lost by closing this — the exact version it proposed is on `dev`.

Root cause of the fan-out is a gap in the shared devkit Renovate preset, filed upstream as **vig-os/devkit#1047**: the preset groups minor/patch for `github-actions` and `pep621`, but the `npm` rule sets no `groupName`, so npm updates fan out one PR per package. With that fixed, this arrives as a single grouped PR next time.
