# [PR 84](https://github.com/vig-os/commit-action/pull/84) chore(github): remove vestigial CODEOWNERS template

## Summary

`.github/CODEOWNERS` was the devkit template with every rule commented out — it matched no
files and served no purpose.

Rather than populating it (the original plan), this **removes the file entirely**: this is a
sole-maintainer repo, and GitHub forbids self-approval, so any real CODEOWNERS entries would
block the maintainer's own PRs behind a code-owner review nobody can provide.

Targets `release/0.3.0` so the removal reaches `main` with the imminent release (Scorecard
scans the default branch) and flows back to `dev` via the main→dev sync.

## Notes

- The `require_code_owner_review: true` toggle in the `main`/`release`/`dev` rulesets becomes
  a no-op without a CODEOWNERS file (harmless, but could be switched off for clarity).
- OpenSSF Scorecard's Branch-Protection check will keep its "codeowners review is not
  required" warning; the other warnings (no approvers, no status checks, admin enforcement)
  are already addressed by the rulesets updated on 2026-07-14. If the alert persists at the
  next scan despite the improved score, dismissing it as won't-fix is the consistent follow-up.
