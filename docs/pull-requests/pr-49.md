## [Comment #2](https://github.com/vig-os/commit-action/pull/49#issuecomment-4967347715) by [@c-vigo](https://github.com/c-vigo)

_Posted on July 14, 2026 at 09:17 AM_

Closing in favour of **#58** (migrate action to ESM).

`@actions/core` v3 is **ESM-only** — `"type": "module"` with an `exports` map exposing only an `"import"` condition, no `"require"`. This repo is CommonJS end to end (`module: commonjs`, ts-jest, ncc → CJS `dist/`), so it cannot be consumed:

```
FAIL src/__tests__/unit/commit-runner.test.ts
  ● Test suite failed to run
    Cannot find module '@actions/core' from 'src/__tests__/unit/commit-runner.test.ts'

npm run bundle → Error: Package path . is not exported from package @actions/core
```

Note `npm run build` (tsc) *passes* — type-checking never executes the `require`, which is why "Lint & Format" was green while "Tests" failed.

Ruled out the `overrides` pins as the cause: removing `"@actions/http-client": "3.0.2"`, reinstalling (confirmed it re-resolved to `4.0.1`), and re-running reproduced the identical failure.

This is not a bump — it needs the coordinated ESM migration in #58, together with #50 and #51.
